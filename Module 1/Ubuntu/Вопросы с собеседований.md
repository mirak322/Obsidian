HTTP:
1) ==**Как работает http протокол**==
Http - это протокол передачи данных, который используется для обмена информацией между клиентом (обычно браузером) и сервером в интернете

a) Клиент отправляет запрос
- Браузер формирует **HTTP-запрос**
    
- Запрос отправляется на **сервер**, на котором расположен сайт
- Запрос содержит:
    - метод (GET, POST и др.)
    - URL
    - заголовки (например, какой браузер, язык и т.д.)
    - тело (для POST/PUT — данные формы и т.п.)

b) Сервер обрабатывает запрос
Сервер:
- получает запрос
- анализирует путь, заголовки и метод
- запускает нужный обработчик (скрипт, программу, отдаёт файл и т.д.)

c) Сервер отправляет ответ
Сервер возвращает **HTTP-ответ**, который содержит:
- статус (200 OK, 404 Not Found, 500 Internal Server Error…)
- заголовки
- тело ответа (HTML, JSON, изображение, видео и др.)

d) Браузер отображает результат

2) ==**Чем отличается HTTP 1.1 от HTTP 2.0?==**

1.Формат передачи данных

HTTP/1.1: текстовый протокол
- Заголовки и структура — текстовые.
- Данные передаются последовательно.
HTTP/2: бинарный протокол
- Всё преобразовано в бинарные фреймы.
- Быстрее парсится и меньше ошибок.

2.Мультиплексирование (главное отличие!)

 HTTP/1.1
 
- Один TCP-соединении = один запрос “за раз”.
- Чтобы ускорить загрузку, браузер открывает несколько соединений (обычно до 6 на домен).
- Проблема — **Head-of-line blocking**: если один запрос завис, стоит вся очередь.

HTTP/2

- В одном TCP-соединении можно отправлять много запросов одновременно.
- Каждый запрос = поток (stream), который идёт параллельно.
- Зависание одного запроса _не блокирует_ остальные.
✔ Более высокая скорость загрузки страниц  
✔ Меньше соединений → меньше накладных расходов

3.Server Push (сервер может отправлять данные сам)
 HTTP/1.1
- Сервер отвечает только на запрос → запрос → ответ → запрос…
HTTP/2
- Сервер может отправить ресурсы заранее.

 4.Приоритеты потоков
 HTTP/2
- Клиент может указать, какие ресурсы важнее (CSS > изображения).
- Браузер быстрее отображает критический контент.
HTTP/1.1 такого не умеет.

5.Скорость http/2 значительно выше

4) ==**Из чего состоит HTTP запрос?==**

1)Стартовая строка (метод, путь, версия протокола)
2)Заголовки(содержит служебную информацию о запросе(тип данных,длина тела))

5) ==**Какие существуют HTTP методы?==**
- **GET** — получить данные
- **POST** — создать ресурс / отправить данные
- **PUT** — полностью обновить ресурс
- **PATCH** — частично обновить ресурс
- **DELETE** — удалить ресурс
- **HEAD** — получить только заголовки ответа (без тела)
- **OPTIONS** — узнать, какие методы поддерживает сервер
- **CONNECT** — установить туннель (обычно для HTTPS)
- **TRACE** — отладочный запрос (редко используется)

5) ==**Какие есть коды ответов в Http? Чем отличается 401 и 403? За что отвечает каждая группа: 1-5xxx==**
**1xx — Информационные**
Запрос получен, продолжаем.  

**2xx — Успех**
Запрос успешно выполнен.  

**3xx — Перенаправление**
Нужен другой URL или доп. действие.  

**4xx — Ошибки клиента**
Проблема в запросе клиента.  

 **5xx — Ошибки сервера**
Сервер не смог обработать запрос.  

**401 Unauthorized**
**Клиент не авторизован.**  
Нужно предоставить корректные данные для входа

 **403 Forbidden**
**Доступ запрещён, даже если авторизация есть.**  
У клиента **нет прав** на доступ.

6) ==**Что такое идемпотентность? Какие методы идемпотентны?==**

Это свойство операции, при котором **повторное выполнение даёт тот же результат**, что и первое

7) ==**PUT vs PATCH==**

PUT — замена ресурса целиком.  
PATCH — изменение части ресурса.

8) ==**Назови основные заголовки HTTP:==**

**Заголовки запроса (Request)**

- **Host** — домен сервера
- **User-Agent** — информация о клиенте
- **Accept** — какие форматы ответа принимает клиент
- **Accept-Language** — язык ответа
- **Content-Type** — тип отправляемых данных
- **Content-Length** — длина тела запроса
- **Authorization** — токен/данные авторизации
- **Cookie** — куки клиента

**Заголовки ответа (Response)**

- **Content-Type** — тип данных в ответе
- **Content-Length** — длина тела ответа
- **Set-Cookie** — установка куки
- **Cache-Control** — правила кеширования
- **Location** — куда перенаправлять (3xx)
- **Server** — информация о сервере

**==OC:==**

1) ==**Назови отличие между многопоточностью и многопроцессностью==**

**Многопоточность (multithreading)**:
- Несколько **потоков внутри одного процесса**
- Все потоки **делят общую память**
- Легче создавать, быстрее переключаются
- Требуют синхронизации (race conditions)
- Подходит для **I/O задач** и лёгкого параллелизма

**Многопроцессность (multiprocessing)**:
- Несколько **процессов**, каждый со **своей памятью**
- Изоляция: сбой одного не ломает другие
- Тяжелее создавать, больше потребляет ресурсов
- Идеально для **параллельных вычислений** (использует все ядра CPU)

2) **==Многопоточность vs ассинхронность==**

- **Многопоточность** - несколько потоков одновременно
- **Асинхронность** - один поток управляет множеством задач **без блокировки**

3) ==**Race Condition==**

это ситуация, когда результат программы зависит от **порядка выполнения потоков или процессов**, из-за чего возникает **непредсказуемое поведение**

4) ==sed, awk, grep==

**grep** - поиск текста
- Используется для **поиска строк, содержащих шаблон**
grep "error" log.txt

**sed** - потоковый редактор
- Используется для **замены, удаления, вставки строк** в файле или потоке текста.
sed 's/old/new/g' file.txt

 **awk** - обработка и анализ текста
- Используется для **парсинга и обработки колонок данных**.

-`grep` → искать
-`sed` → заменять/редактировать
-`awk` → анализировать/выбирать поля

5) ==Расскажи про систему прав==

Основные сущности
1. **Владелец (user)** — пользователь, которому принадлежит файл
2. **Группа (group)** — группа пользователей, имеющая специальные права
3. **Остальные (others)** — все остальные пользователи

r(read) - читать
w (write) - запись или изменение
x (execute) - выполнение файла или вход в каталог

-`rwx` → владелец: читать, писать, запускать
-`r-x` → группа: читать, запускать
-`r--` → остальные: только читать

Для изменения прав используется команда chmod

chmod 775 file.txt


==Что происходит после ввода адреса в адресную строку и нажатия на Enter?==

- **Преобразование имени в IP (DNS)**
- Браузер спрашивает DNS-сервер, какой IP у `example.com`.

- **Установка соединения с сервером**
- TCP/TLS (если HTTPS) соединение с сервером по IP и порту 80/443.

- **Отправка HTTP-запроса**
- Браузер отправляет `GET /` и заголовки (User-Agent, Cookies и др.).

- **Обработка запроса сервером**
- Сервер формирует ответ: HTML, CSS, JS, изображения.

- **Получение и рендеринг страницы**
- Браузер получает ответ, парсит HTML, делает дополнительные запросы к ресурсам (CSS, JS, картинки) и строит страницу.

- **Выполнение скриптов и отображение**
- JS выполняется, страница становится интерактивной.
